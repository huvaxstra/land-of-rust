>> مرحله ۲: راهروی یک‌نفره (Mutex) >> Step 2: One-Way Hallway (Mutex)

برای تغییر یک داده‌ی مشترک توسط چند مورچه، باید از درگیری (Data Race) جلوگیری کنیم. `Mutex` مثل یک درِ قفل‌دار است. هر مورچه باید کلید را بگیرد، کارش را بکند و بعد قفل را باز کند. با کدام متد کلید قفل را می‌گیریم؟

To modify shared data from multiple ants, we must prevent Data Races. A `Mutex` is like a locked door. An ant must get the key, do the work, and release it. Which method grabs the lock key?

use std::sync::Mutex;

let food = Mutex::new(0);
// Ant requests access
let mut vault = food.---().unwrap();
*vault += 1; // Safe modification!
