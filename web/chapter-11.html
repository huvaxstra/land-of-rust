Chapter 11: Flight Simulator (Writing Tests) 🧪

> قبل از پرواز واقعی، باید همه سیستم‌ها را در «شبیه‌ساز پرواز» امتحان کنیم.

> در برنامه‌نویسی به این کار نوشتن **تست (Test)** می‌گویند. ما کدهای کوچکی می‌نویسیم که بررسی می‌کنند آیا کدهای اصلی درست کار می‌کنند یا نه.

برای اینکه به کامپایلر بگوییم یک تابع فقط برای تست کردن است، چه کلمه‌ای را داخل `#[ ]` بالای آن می‌نویسیم؟

> Before the real flight, we must try all systems in the "Flight Simulator".

> In programming, this is called writing **Tests**. We write small codes to check if our main code works correctly.

To tell the compiler a function is only for testing, what word do we write inside the `#[ ]` above it?

#[---]
fn check_engine() {
    // Test code here...
}

داخل یک تست، ما باید نتیجه را با چیزی که انتظار داریم مقایسه کنیم. اگر ۲+۲ را محاسبه کردیم، باید مطمئن شویم جواب دقیقاً ۴ است.

کدام ماکرو (وِرد جادویی) برای مقایسه دقیق و بررسی «برابر بودن» دو مقدار استفاده می‌شود؟

Inside a test, we must compare the result with what we expect. If we calculate 2+2, we must ensure the answer is exactly 4.

Which macro (magic word) is used to strictly compare and check if two values are "equal"?

#[test]
fn math_works() {
    ---!(2 + 2, 4);
}

>> مرحله ۲: تست دکمه خودتخریبی! >> Step 2: Testing the Self-Destruct Button!

بعضی وقت‌ها برنامه ما *باید* خطا بدهد (مثل وقتی می‌خواهیم عدد را بر صفر تقسیم کنیم). چطور به کامپایلر بگوییم: «اگر این تابع منفجر شد (panic)، یعنی تست با موفقیت پاس شده است!»؟

Sometimes our program *should* error out (like dividing by zero). How do we tell the compiler: "If this function explodes (panics), it means the test passed successfully!"?

#[test]
#[---(expected = "Divide by zero")]
fn test_bad_math() {
    divide(10, 0); // Boom! 💥
}
