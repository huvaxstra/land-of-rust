Chapter 14: The Toy Supermarket (Crates.io) 🛒

> لازم نیست همه برنامه‌ها را خودت از صفر بنویسی! به جای آن، می‌توانیم ابزارهای آماده را از فروشگاه **crates.io** قرض بگیریم.

> برای اینکه کارگو (Cargo) بداند چه بسته‌ای را می‌خواهیم، باید اسم آن را در یک فایل خاص بنویسیم.

برای اضافه کردن یک ابزار جدید به پروژه، آن را زیر کدام کلمه در فایل `Cargo.toml` می‌نویسیم؟

> You don't have to write everything from scratch! Instead, we can borrow ready-made tools from **crates.io**.

> For Cargo to know which package we want, we must write its name in a specific file.

To add a new tool to our project, under which word in `Cargo.toml` do we write it?

# Cargo.toml
---
rand = "0.9.0"
ferris-says = "0.3"
[crates]
[imports]
[dependencies]
[packages]
>> مرحله ۱: وارد کردن بسته به کد >> Step 1: Bringing the Crate into Code

وقتی کارگو ابزار `rand` را دانلود کرد، باید به کامپایلر بگوییم که می‌خواهیم از آن در فایلمان استفاده کنیم.

از کدام کلمه کلیدی برای آوردن توابع یک Crate به داخل برنامه استفاده می‌کنیم؟

Once Cargo downloads the `rand` tool, we need to tell the compiler we want to use it in our file.

Which keyword do we use to bring a Crate's functions into our program?

--- rand::Rng;

fn main() {
    let num = rand::thread_rng().gen();
}
import
include
use
get
>> مرحله ۲: ساختن اسباب‌بازی خودمان >> Step 2: Making Our Own Toy

ما هم می‌توانیم کدهایمان را به عنوان یک Crate برای بقیه منتشر کنیم! برای این کار باید پروژه‌مان را به جای برنامه اجرایی، به عنوان یک **کتابخانه** بسازیم. کدام پرچم (Flag) در دستور `cargo new` این کار را می‌کند؟

We can also publish our code as a Crate for others! To do this, we must build our project as a **Library** instead of an executable. Which flag in `cargo new` does this?

$ cargo new pig_latin ---
Created library `pig_latin` package
--bin
--lib
--crate
--toy
