فصل ۳: دستور پخت کیک شکلاتی 🍰

Chapter 3: Space Chocolate Cake Recipe 🍰

> در برنامه‌نویسی وقتی یک کار تکراری داریم، به جای نوشتن دوباره کدها، از تابع (Function) استفاده می‌کنیم.

> تابع دقیقا مثل یک «دستور پخت جادویی» است. یک بار آن را می‌نویسیم و هر چند بار که بخواهیم از آن کیک (خروجی) می‌گیریم!

// ساخت یک ماشین پخت کیک
fn bake_cake() {
    println!("Baking a chocolate cake! 🍰");
}
// حالا می‌تونیم هزار بار صداش بزنیم:
bake_cake();

> When we have repetitive tasks, instead of rewriting code, we use a Function.

> A function is like a "magic recipe". We write it once, and we can get cakes (outputs) from it as many times as we want!

// Creating a cake baking machine
fn bake_cake() {
    println!("Baking a chocolate cake! 🍰");
}
// Now we can call it a thousand times:
bake_cake();
>> مرحله ۱: خروجی ماشین (Return Values) >> Step 1: Machine Output (Return Values)

توابع می‌توانند خروجی داشته باشند (مثلاً یک تابع، دو عدد را جمع کند و جواب را پس بدهد).

در زبان راست، برای نشان دادن اینکه این تابع قرار است یک عدد (i32) را پس بدهد، از یک فلش (Arrow) استفاده می‌کنیم.

شکل درست این فلش کدام است؟ آن را در جای خالی قرار بده:

Functions can return an output (like adding two numbers and giving back the result).

In Rust, to show that a function will return a number (i32), we use an Arrow symbol.

Which one is the correct arrow? Place it in the blank:

fn add_sugar(a: i32, b: i32) --- i32 {
    a + b // No semicolon means return!
}
