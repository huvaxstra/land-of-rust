# 🤝 Contributing to "The Land of Rust"

Thank you for wanting to help make Rust accessible to young learners! 🦀✨

This project welcomes contributions from educators, developers, parents, and especially curious kids. Here's how you can help.

## 🌟 Ways to Contribute

| Type | Examples | How to Start |
|------|----------|--------------|
| 🐛 Bug Reports | Typos, broken code examples, unclear explanations | Open a [GitHub Issue](#) with details |
| 💡 Content Ideas | Better metaphors, new exercises, cultural adaptations | Start a [Discussion](#) or open an issue |
| 🌍 Translations | Translate chapters or teacher guide to your language | Open an issue first to coordinate efforts |
| 👩‍🏫 Teaching Feedback | Share how a chapter worked (or didn't) with your students | Comment on issues or email directly |
| 🎨 Illustration Prompts | Suggest AI image prompts for chapter artwork | Add to `/assets/prompts/` via PR |
| 🔧 Technical Improvements | Fix code examples, improve tooling, enhance accessibility | Fork, branch, and submit a PR |

## 🧭 Contribution Guidelines

### For Content & Pedagogy
- Keep language **simple, warm, and age-appropriate** (target: ages 9+)
- Avoid assuming prior programming knowledge
- Prefer **concrete metaphors** over abstract explanations
- Code examples must:
  - Compile with **Rust 1.75+** and **edition 2021**
  - Include comments for non-obvious lines
  - Avoid unsafe code unless explicitly teaching it (Ch 19)

### For Code & Technical PRs
- Follow standard Rust formatting (`cargo fmt`)
- Ensure examples are copy-paste runnable
- Add tests if modifying core example logic
- Update relevant documentation

### For Translations
- Maintain the story tone and Ferris's personality
- Preserve code examples in English (with Persian/RTL comments if needed)
- Coordinate with maintainers to avoid duplicate efforts

## 🔄 Pull Request Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-idea`
3. **Make your changes** following the guidelines above
4. **Test locally**: Ensure examples compile and run
5. **Submit a PR** with:
   - Clear title and description
   - Reference to related issue (if any)
   - Screenshots for visual/content changes (if helpful)

> 💡 For major changes (new chapter, structural overhaul), **please open an issue first** to discuss with maintainers.

## 🧒 Working with Content for Children

Because this book is designed for young learners:
- Use inclusive, encouraging language
- Avoid stereotypes in examples or illustrations
- Prioritize emotional safety: no scary error messages, frame mistakes as learning
- When in doubt, ask: "Would this make a 10-year-old feel capable and curious?"

## 📜 Code of Conduct

This project follows the [Rust Code of Conduct](https://www.rust-lang.org/policies/code-of-conduct).

Key principles:
- Be respectful and empathetic
- Welcome beginners – everyone starts somewhere
- Assume good intent
- If you see something, say something (contact maintainers)
